# しょうの助シフト

Python、Streamlit、SQLite を使ったシフト希望提出アプリです。  
バイトメンバーは月ごとの希望開始時間を入力でき、管理者は必要人員を設定して自動シフトを作成できます。

## アプリの概要

- メンバー用画面
  - 名前を選択
  - 年度を選択
  - 月を選択
  - 前半 / 後半を選択
  - 選んだ半月分のカレンダー上で各日付の希望開始時間を入力
  - `×` を選ぶと出勤不可
  - 提出内容を SQLite に保存
- 管理者用画面
  - メンバー名を登録
  - 登録済みメンバー名を削除
  - 前半 / 後半ごとにカレンダー上で時間帯ごとの必要人数を設定
  - 希望シフト一覧を確認
  - 必要人員と希望から自動シフトを作成
  - CSV をダウンロード

## 使い方

### 1. 希望シフト提出

- サイドバーで `希望シフト提出` を選びます
- 名前を選びます
- 年度と月を選びます
- 前半または後半を選びます
- 各日付で希望開始時間を選びます
- 出勤できない日は `×` を選びます
- `リセット` を押すと、その半月の入力をすべて `×` に戻せます
- `希望シフトを保存` を押します

補足:
- `17:30` を希望した場合は、`17:30` 以降で勤務可能という扱いです

### 2. 管理者画面

- サイドバーで `管理者画面` を選びます
- 必要に応じてメンバー名を登録または削除します
- 年度と月を選びます
- 前半または後半を選びます
- 各日付について `17:00 / 17:30 / 18:00 / 18:30 / 19:00` の必要人数を入力します
- `必要人員カレンダーを保存` を押します
- `シフトを作成` を押すと、縦軸が名前、横軸が日付のシフト表を自動生成します

### 3. 自動シフト作成のルール

- 必要人員を優先して、早い時間帯の不足から順に割り当てます
- メンバーは希望開始時間以降の時間帯に勤務可能として扱います
- 例: `17:30` 希望なら `17:30 / 18:00 / 18:30 / 19:00` に勤務可能です
- 必要人数を満たせない時間帯がある場合は警告が表示されます

### 4. 半月の区切り

- 前半: 1日〜15日
- 後半: 16日〜月末

## 補足

- データベースファイルは `shifts.db` です
- 初回起動時に必要なテーブルが自動作成されます
